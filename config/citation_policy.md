# 인용 정책 (Citation Policy)

이 정책은 **Stage 1(리서치) 산출물의 모든 주장**에 적용됩니다.
Stage 2 verifier는 이 정책 위반을 거르고, Stage 3 writer는 정책을 통과한 주장만 사용해야 합니다.

---

## 1. 모든 주장 = 출처 + 원문 인용 1:1

주장 1개당 다음 3요소가 **반드시 함께** 있어야 합니다:

| 요소 | 내용 |
|---|---|
| **주장 (claim)** | 한 문장으로 압축된 명제 |
| **출처 (source)** | 판례번호 · 서지정보 · 공식 URL |
| **원문 (verbatim)** | 출처에서 그대로 인용한 문장 (생략·축약 금지) |

## 2. 인용 없는 주장 = 출력 금지

원문 인용을 첨부할 수 없는 주장은 **출력 자체를 금지**합니다.

- ❌ "~로 알려져 있다"
- ❌ "~가 통설이다"
- ❌ "~라는 견해가 유력하다" (출처 없이)

이런 무출처 진술은 환각 위험이 가장 높은 패턴입니다.

## 3. URL 검증 단계 표기

| 상황 | 표기 |
|---|---|
| URL을 직접 확인했고 접근 가능 | URL 그대로 |
| URL이 불확실하거나 변동 가능 | `URL_UNVERIFIED` 표시 + 출처명 |
| URL이 없는 인쇄물 (책 등) | ISBN 또는 출판사·발행연도 |

## 4. 한국 판례 인용 형식

- **대법원:** `대법원 2020. 3. 26. 선고 2019다12345 판결`
- **헌법재판소:** `헌법재판소 2019. 12. 27. 선고 2018헌마730 결정`
- **하급심:** `서울고등법원 2021. 5. 14. 선고 2020나98765 판결`
- **법령:** `민법 제103조` (조·항·호까지 명시)

## 5. 학설·주석서 인용 형식

`저자명, 『책명』, 출판사, 출판연도, 페이지` 형식.
예: `곽윤직, 『민법총칙』, 박영사, 2012, 234면`

## 6. 위반 시 처리

- Stage 1: 에이전트가 자체적으로 위반 주장을 출력에서 제외
- Stage 2 verifier: 위반 발견 시 해당 주장을 `verified_claims.md`에서 제외하고 사유 기록
- Stage 3 writer: 검증 통과 자료만 받으므로 추가 검사 불필요
